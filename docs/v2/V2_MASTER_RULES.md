# BSDC FRONTEND V2 — MASTER RULES

Status: LOCKED

---

## Core Definition

Frontend V2 is NOT:

- a website
- a landing page
- a section stack
- a slideshow
- a gallery
- a dashboard
- a collection of cards

Frontend V2 IS:

- a continuous spatial world
- a camera-driven experience
- an engineering map
- a navigable route through BSDC content

---

## World Rules

There is ONE world.

There are NOT multiple pages.

There are NOT multiple scenes.

There is ONE continuous route from:

Hero
→ About
→ Certificates
→ Partners
→ Services
→ Projects
→ Contact
→ Footer

---

## Camera Rules

Scroll controls camera movement.

Camera is the primary navigation mechanism.

Content must never switch.

Content must never be replaced.

The camera changes what the user perceives.

---

## Forbidden Patterns

FORBIDDEN:

- slideshow behavior
- gallery behavior as primary experience
- fade between sections
- page transitions
- viewport replacement
- card grids as primary layout
- section stacking
- carousel-first UX
- generic corporate layouts
- WordPress-like layouts

---

## Node Rules

Nodes are world objects.

Nodes are NOT pages.

Nodes are NOT sections.

Nodes are NOT slides.

Nodes exist permanently in the world.

---

## CMS Rules

Use existing CMS only.

NO schema changes.

NO new Prisma models.

NO CMS restructuring.

Mappings are fixed:

Settings → Hero
About → About
Certificates → Certificates
Partners → Partners
Services → Services
Projects → Projects
Contact → Contact

---

## Motion Rules

GSAP only.

Motion must feel:

- physical
- heavy
- deliberate
- spatial

Motion must NOT feel:

- flashy
- sci-fi
- game-like
- decorative

---

## Success Condition

A successful V2 must feel like:

"an interactive engineering exhibition"

and never:

"a company website".
## ABSOLUTE NAVIGATION RULE CHANGE

The "treasure map / visible route line / node path overlay" is permanently removed.

It MUST NOT exist as:

- visible navigation layer
- always-on SVG path
- background route guide
- map-like structure
- node connection visualization

---

## NEW PRINCIPLE

There is NO visible map.

There is ONLY experience traversal.

---

## FORMER MODEL (INVALID)

- visible route line
- node-to-node map display
- path overlay
- treasure map logic

This model is explicitly forbidden.

---

## CURRENT MODEL (VALID)

Navigation is implicit.

The user does NOT see the system structure.

The user only experiences:

- arrival
- immersion
- departure

NO structural diagram is visible at any time.
